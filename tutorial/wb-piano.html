Waterbear Piano Step 1
Goal: Make a piano in Waterbear!
Step 1 : Draw the piano on the screen.

Instructions: Let's start by drawing the piano! We will draw ours with one black rectangle for a base, and seven white rectangles for keys. It might help to put labels on the keys to indicate what notes you will want them to play!

Note: Waterbear has some built in notes, from A to G, with the sharps and flats between.

You can draw things overtop one another. The stroke shape block will give the shape a border. It doesn't matter where the screen the piano is drawn

Blocks to use:
Here are the blocks you'll need:
fill style stroke style fill the shape stroke the shape rectangle at width height orientation fill text x y font x y
Strategy Hints:
Make sure do draw the shapes in order: The back of the piano has to be drawn before the keys, or it will draw overtop of them!
The "stroke color" is the border color, and "fill color" is the inside. If you give the keys a black border it seems like there's space between them!
Make sure not to write black text on the piano base, or white text on the keys

Waterbear in Space Step 2
Step 2 : Add the notes

Instructions: Let's make some music! We are going to use the keyboard like, well, a keyboard! The "when key pressed" block waits until a button on the keyboard is pressed, then executes the blocks inside. Pick a key on the keyboard to each key on the piano, and when that key is pressed, play the right note!

You can use the following notes:

Waterbear Piano Step 3
Step 3 : Let's add some animation

Instructions: We didnt draw that piano for nothing! When we press a key, let's make the corresponding piano keychange colour for a second. You can do this by drawing a different color rectangle (we will use grey) over the key that was pressed, and then redrawing the key in white.

Blocks to use:
Here are the blocks you'll need:
fill style stroke style fill the shape stroke the shape rectangle at width height orientation fill text x y font x y
Strategy Hints:
Did you redraw the text as well as the rectangle for the key?
Look back at the blocks from the original drawing, the rectangles for the animation will have the same size and location of the originals!
Make sure you are not just drawing the same rectangle, no matter what key is pressed.
You will have to change the fill color multiple times

Actual output:
